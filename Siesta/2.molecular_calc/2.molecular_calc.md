## 2. 분자 계산 개요 및 실습 목표

이 장에서는 SIESTA 5.x 를 이용한 **비주기계(분자) 계산의 기본 workflow** 를 연습한다. 정답 파일을 그대로 따라 치는 것이 아니라, "초기 구조를 준비하고 계산한 뒤 결과를 확인하는 흐름"을 익히는 것이 목표다.

### 실습 목표

- **$CH_4$**: 구조 최적화를 수행하고, **basis 크기에 따라 에너지와 구조가 어떻게 달라지는지** 확인.
- **$H_2O$**: 구조 최적화를 수행하고, **전자 밀도(charge density)** 를 저장해 결합 전자 분포를 살펴봄.
- **$NH_3$ (과제)**: 앞의 두 예제를 바탕으로, **직접 구조를 수정하고 계산 항목을 선택해 보는 연습**.

### 분자 계산의 기본 workflow

1. 분자 구조 준비 (`STRUCT.fdf`)
2. 계산 옵션 설정 (`RUN.fdf`, `BASIS.fdf`, `KPT.fdf`)
3. 구조 최적화 실행 (`MD.TypeOfRun CG`)
4. 최적화 결과 구조 확인 (`<Label>.XV`, `<Label>.ANI`, `<Label>.xyz` 등)
5. 후처리 계산 수행 (basis test, 전자 밀도 시각화 등)
6. 결과 비교 및 해석

> 이 장의 목표는 "정답 구조를 외우는 것"이 아니라,  
> **초기 구조 → 최적화 → 결과 검증 → 추가 분석** 의 흐름을 익히는 것이다.

## 3. 공통 준비: 입력 파일 구성과 실행 방법

분자 계산($CH_4$ / $H_2O$ / $NH_3$) 은 모두 비슷한 폴더 구조와 입력 파일 구성을 사용한다.

### 3.1 입력 파일 구성

SIESTA 계산에는 일반적으로 다음 파일들이 필요하다.

- `RUN.fdf`: 계산 종류(SCF, 구조 최적화 등), 수렴 조건, 출력 옵션, XC 함수
- `STRUCT.fdf`: 원자 좌표와 셀(cell) 정보
- `KPT.fdf`: k-point 설정. 분자 계산에서는 보통 `1×1×1` 을 사용
- `BASIS.fdf`: basis 관련 설정 (`PAO.BasisSize` 등)
- `*.psml`: 원자별 PSML 슈도포텐셜 파일 (예: `C.psml`, `H.psml`, `O.psml`, `N.psml`)

`RUN.fdf` 안에서 `%include` 로 STRUCT/BASIS/KPT 를 가져오는 구조이므로, 실제 실행 시 SIESTA 가 읽는 것은 `RUN.fdf` 하나다.

> 분자 계산이라도 `STRUCT.fdf` 안에는 충분히 큰 셀(box)을 넣어 주는 것이 중요하다.  
> 셀이 너무 작으면 주기경계조건 때문에 옆 셀의 분자와 원하지 않는 상호작용이 생길 수 있다.

### 3.2 구조 파일 준비 (`STRUCT.fdf`)

분자 계산의 첫 단계는 **초기 구조를 준비하는 것**이다. 튜토리얼 폴더에 예제 구조가 들어 있더라도, 직접 만들어 보는 과정을 한 번 익혀 두면 이후 다른 분자나 물질을 다룰 때 도움이 된다.

원하는 분자 구조는 Materials Project 같은 외부 데이터베이스에서 받아 변환해 사용할 수 있다. 가장 간단한 방법은 ASE 를 이용해 `POSCAR`/`CIF` 를 fdf 로 변환하는 것이다.

```python
# poscar_to_fdf.py
from ase.io import read, write
atoms = read("POSCAR")      # 또는 read("struct.cif")
write("STRUCT.fdf", atoms, format="siesta-fdf")
```

```bash
$ python poscar_to_fdf.py
```

#### 예시 `STRUCT.fdf` (CH$_4$, 의도적 비대칭 입력)

분자 계산에서는 셀을 크게 잡고 (예: 10 Å 큐브) 분자를 셀 중앙 근처에 두는 방식으로 시작한다.

```fdf
NumberOfAtoms    5
NumberOfSpecies  2

%block ChemicalSpeciesLabel
 1 6 C
 2 1 H
%endblock ChemicalSpeciesLabel

LatticeConstant 10.000000000 Ang

%block LatticeVectors
    1.000000000     0.000000000     0.000000000
    0.000000000     1.000000000     0.000000000
    0.000000000     0.000000000     1.000000000
%endblock LatticeVectors

AtomicCoordinatesFormat ScaledCartesian

%block AtomicCoordinatesAndAtomicSpecies
 0.4991728  0.4991728  0.4992459  1  1
 0.6028726  0.4747901  0.4680767  2  2
 0.4747901  0.6028726  0.4680767  2  3
 0.4908115  0.4908115  0.6096102  2  4
 0.4285072  0.4285072  0.4511578  2  5
%endblock AtomicCoordinatesAndAtomicSpecies
```

### 3.3 PSML 슈도포텐셜 준비

SIESTA 5.x 는 **PSML (XML 기반) 슈도포텐셜**을 그대로 읽을 수 있다 (확장자만 보고 자동 인식). 별도의 atom 변환 단계가 필요하지 않다.

`pseudos/psml/` 폴더에 미리 준비된 [Pseudo-Dojo](https://www.pseudo-dojo.org/) NC-SR PBE standard set 의 `C.psml`, `H.psml`, `O.psml`, `N.psml` 가 들어 있다. 다른 원소가 필요하면 같은 site 의 `nc-sr-04_pbe_standard_psml.tgz` 에서 받아 풀어 두면 된다.

```bash
$ wget https://www.pseudo-dojo.org/pseudos/nc-sr-04_pbe_standard_psml.tgz
$ tar xzf nc-sr-04_pbe_standard_psml.tgz <원소>.psml
```

> **주의**: PSML 은 생성 시 사용된 XC 함수가 메타데이터에 박혀 있다. `RUN.fdf` 에서 `XC.functional GGA` + `XC.authors PBE` 를 명시해 PSML 과 일치시켜야 SIESTA 가 거부하지 않는다.

`run.sh` 안에서 `pseudos/psml/<원소>.psml` 를 작업 디렉토리로 심볼릭 링크하는 부분이 자동으로 수행된다.

### 4.1 CH4: basis 크기에 따른 에너지/구조 변화(수렴성) 확인

#### 4.1.1 CH4 분자 구조 확인

이번 실습에서는 일부러 약간 어긋난 구조를 넣고, 구조 최적화를 통해 올바른 형태로 돌아가는지 확인해 본다. $CH_4$ 는 네 개의 C-H 결합 길이가 모두 같아야 하는 대표적인 분자다.

입력 구조를 한 번 눈으로 확인해 보자.

```bash
$ python scripts/render_structure.py ch4_input_in summer\ school/CH4/STRUCT.fdf
```

`STRUCT.fdf` 를 보면 한 H 의 위치가 다른 H 들과 약간 비대칭이라는 것을 확인할 수 있다.

#### 4.1.2 구조 최적화 실행 및 결과 확인

```bash
$ cd summer\ school/CH4
$ bash run.sh
```

`RUN.fdf` 의 핵심 항목:

- `MD.TypeOfRun CG` + `MD.Steps 300` — 충분히 최적화가 진행되게 한다.
  > **5.x 변경점**: 4.1.5 의 `MD.NumberCGsteps` 가 5.x 에서는 `MD.Steps` 로 통합되었다. 이전 키워드는 무시되고 기본값(0)이 적용되므로 반드시 새 이름을 사용해야 한다.
- `MD.MaxForceTol 0.01 eV/Ang` — 종료 기준.
- `PAO.BasisSize DZP` — 기본 basis (검증 단계에서는 SZ 부터 시작해도 된다).

계산이 끝나면 `CH4.out` 에서 다음과 같은 라인을 볼 수 있다.

```

```

구조 변화는 `CH4.ANI` (모든 step 의 xyz) 또는 `CH4.XV` (최종 좌표) 로 확인.

```bash

```

![ch4_optimized_structure](img/ch4_optimized_structure.png)

입력 구조에서는 결합 길이가 약 `0.001 Å` 정도 차이 났지만, 최적화 후에는 차이가 더 작아진다. $CH_4$ 의 자연스러운 정사면체 대칭으로 수렴했음을 확인할 수 있다.

#### 4.1.3 CH4 분자 basis 수렴성 확인

구조 최적화가 끝났다면, 이번에는 사용한 basis 가 충분한지 확인한다. 보통은 최적화된 구조를 기준으로 `PAO.BasisSize` 만 바꿔 가며 계산한다. 미리 준비된 `summer school/CH4_basis_sweep/` 폴더에 **SZ / DZ / TZ / DZP** 네 케이스가 들어 있다.

```bash
$ cd summer\ school/CH4_basis_sweep
$ for d in SZ DZ TZ DZP; do (cd "$d" && bash run.sh); done
```

> **DZDP 가 빠진 이유**: SIESTA 5.4 + Pseudo-Dojo PSML 조합에서 C 의 2p 분극 orbital 에 대한 2nd-zeta 가 너무 degenerate 하여 자동 split (split-by-tail-norm) 이 실패한다. DZDP 를 쓰려면 `PAO.Basis` 블록을 명시해 C 의 분극 channel 을 직접 정의해야 한다. 학생 과제로 적합.

결과는 `scripts/basis_convergence.py` 로 한꺼번에 정리한다.

```bash
$ cd wiki\ update
$ python scripts/basis_convergence.py
```

출력으로 `img/ch4_basis_convergence.png` 와 `img/ch4_basis_convergence.csv` 가 생성된다.

![ch4_basis_convergence](img/ch4_basis_convergence.png)

`img/ch4_basis_convergence_table.md` 에 markdown 표 단편도 자동으로 떨어지므로, 본 페이지에 그대로 복사해 넣을 수 있다.

Reference: Handy, Murray & Amos, J. Phys. Chem. **97**, 4392-4396 (1993). C-H 1.087 Å, H-C-H 109.47°.

#### 실측값 (SIESTA 5.4.2 + Pseudo-Dojo PBE)

|  | SZ | DZ | TZ | DZP |
| --- | --- | --- | --- | --- |
| `Bond length [Å]` | 1.194 | 1.110 | 1.111 | 1.113 |
| `Bond angle [°]`  | 109.47 | 109.47 | 109.47 | 109.47 |
| `E_KS [eV]`       | −224.41 | −227.58 | −227.66 | −227.99 |

`SZ` 는 거칠고(reference 대비 ~0.1 Å 길게 나옴), `DZ` 이상부터 빠르게 안정된다. **총에너지 기준 SZ→DZP 차이가 3 eV 가량으로 크게 줄어**드는 것이 더 또렷한 signal 이다. 실제 계산에서는 정확도와 비용을 함께 고려해 `DZP` 를 기본으로 선택하는 것이 보통이다.

> **5.x 변경점 메모**: 4.1.5 에서 같은 표를 만들면 PSF 기반이라 절대값이 약간 다르게 나온다. 5.x + Pseudo-Dojo PSML 조합은 일반적으로 결합 길이가 ~0.01 Å 짧게 나오는 경향이 있다 — 이는 정상이며 (NC-SR PBE 의 특성), 학생들이 두 표를 비교했을 때 당황하지 않도록 안내가 필요하다.

### 4.3 NH3 개별 실습 과제

이제 앞에서 했던 과정을 $NH_3$ 에 직접 적용해 본다. 처음부터 완전히 새로운 작업을 하는 것이 아니라, `CH4` 와 `H2O` 에서 했던 절차를 스스로 다시 조합하는 연습이라고 생각하면 된다.

권장 순서:

1. `summer school/NH3/STRUCT.fdf` 에서 한 원자의 위치를 조금 바꿔 "약간 틀어진 초기 구조" 를 만든다.
2. `bash run.sh` 로 구조 최적화를 실행해 안정한 구조로 수렴하는지 확인한다 (N-H ≈ 1.01 Å, H-N-H ≈ 107°).
3. `BASIS.fdf` 에서 `PAO.BasisSize` 를 바꿔 basis 수렴성을 간단히 비교해 본다.
4. 필요하면 `RUN.fdf` 에 `SaveRho T`, `SaveDeltaRho T` 를 켜고 `scripts/plot_rho_slice.py` 의 `DEFAULT_DIR` 를 `NH3` 로 바꿔 전자 밀도까지 시각화한다.

전자 밀도 그림에서 isosurface (또는 컬러 범위) 가 너무 진하거나 약하면 `plot_rho_slice.py` 의 `extent_ang` 또는 cmap 범위(`vmax` 설정 부분)를 조정해 보면 된다. 원본 튜토리얼의 VESTA isosurface level 0.007 정도의 표시는, 이 스크립트에서는 LogNorm 의 `vmax` 를 0.007 근처로 잡으면 비슷한 인상을 준다.

## 부록 A. 빌드 명령 요약 (관리자용)

이미 빌드되어 있는 SIESTA 5.4.2 를 사용한다면 이 절은 건너뛰어도 된다.

```bash
# 1) 의존성 (CMake, libxc 6, NetCDF-Fortran, HDF5)
conda create -n siesta-deps -c conda-forge -y cmake libxc=6 netcdf-fortran hdf5

# 2) SIESTA 5.4.2 소스
git clone --branch 5.4.2 --recurse-submodules \
  https://gitlab.com/siesta-project/siesta.git siesta-5.4.2

# 3) 환경 + CMake configure
source /home/users2/cha/miniconda3/etc/profile.d/conda.sh
conda activate siesta-deps
source /home/shared/intel/oneapi/setvars.sh
export PKG_CONFIG_PATH="$CONDA_PREFIX/lib/pkgconfig:$PKG_CONFIG_PATH"

cd siesta-5.4.2 && mkdir _build && cd _build
FC=mpiifort CC=mpiicc cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_INSTALL_PREFIX="$(cd .. && pwd)/_install" \
  -DSIESTA_TOOLCHAIN=intel \
  -DSIESTA_WITH_MPI=ON -DSIESTA_WITH_LIBXC=ON -DSIESTA_WITH_NETCDF=ON \
  -DSIESTA_WITH_FLOOK=OFF -DSIESTA_WITH_WANNIER90=OFF \
  -DSCALAPACK_LIBRARY="-L$MKLROOT/lib/intel64 -lmkl_scalapack_lp64 -lmkl_blacs_intelmpi_lp64" \
  -DNetCDF_ROOT=$CONDA_PREFIX

# 4) build + install
cmake --build . -j 8
cmake --install .

# 5) 검증
ctest -L simple --output-on-failure
$SIESTA_HOME/bin/siesta --version    # 5.4.2 출력 확인
```
